### 🚀 结构化开发计划 (Structured Development Plan)

#### Phase 1：基建与数据模型 (Infrastructure & Data Schema)

* [x] **Task 1.1：设计并初始化本地 SQLite 核心配置表**
* 创建 `search_strategies` 与 `blacklists` 表。
* 预置零硬编码的高频工作区规则（例如，优先配置解析 `~/Documents/Douban_Lists` 目录下的 Markdown 文件，以及 `~/Library/Application Support/Steam` 目录下的相关存档数据，并设定 `max_size_mb` 为 50MB 阈值）。


* [x] **Task 1.2：构建本地 FTS5 倒排索引实体与云端主库模型**
* 初始化本地 `file_index` 表（包含物理标识、扩展名、修改时间等）并挂载 FTS5 虚拟表以支持极速正则/内容匹配。
* 利用 SQLModel 与 Pydantic V2 构建云端 PostgreSQL 的用户订阅状态、全局同义词库及端云配置同步接口的数据契约。



#### Phase 2：后端核心逻辑与 API (Backend Services & APIs)

* [x] **Task 2.1：开发 Rust 端资源嗅探器与权限引导引擎**
* 封装 macOS 原生 API 测试“完全磁盘访问权限 (Full Disk Access)”，若未授权则触发阻断式沙盒模式引导。
* 实现后台电量与负载嗅探轮询：当设备未插电且电量低于 20%，或 CPU 温度 $> 75^\circ\text{C}$ 时，强制挂起内容建库任务，仅保留文件名更新。
* 当前进展：已初始化 Rust core crate，完成配置化资源策略、Full Disk Access 探测路径、pmset 电量读取解析、CPU 温度读取失败兜底、权限引导状态、低电量/高温挂起判定、资源轮询 provider 边界与 Tauri command 形态函数；`cargo test` 已通过。


* [x] **Task 2.2：构建本地“双层漏斗”遍历与 FSEvents 监听队列**
* 实现首层极速扫描：利用 Rust `jwalk` 多线程库或原生 APFS 读取器，秒级完成全盘文件名遍历入库。
* 挂载 `FSEvents` 监听特定 Opt-in 目录，将增删改动作推入异步解析队列（剥离大文件与加密文件）。
* 当前进展：已完成配置化 indexer policy、标准库递归文件系统扫描 provider、首层文件名元数据漏斗、排除路径过滤、大文件/加密文件降级为仅文件名、Opt-in 事件适配与解析队列，以及 macOS FSEvents stream 绑定入口；`cargo test` 已通过。


* [x] **Task 2.3：部署云端 FastAPI NL2DSL 引擎与语义缓存**
* 基于 `/prompts/nl2dsl/v1.json` 构建结构化 Prompt 管线，强制 LLM 输出严格校验的查询 DSL JSON。
* 接入本地向量缓存层：拦截相似度 $> 0.92$ 的历史搜索，直接返回缓存 DSL；对于真实 API 调用，强制使用 SSE 向前端推送中间解析状态。
* 当前进展：已完成配置化 Prompt 加载、严格 Pydantic DSL 校验、FastAPI SSE 事件骨架、本地向量缓存、缓存优先拦截、LLM Provider 输出 JSON 严格校验契约，以及配置化 HTTP LLM Provider 入口。



#### Phase 3：前端视图与交互闭环 (Frontend UI & Interactions)

* [x] **Task 3.1：搭建 Raycast 风格极简搜索基座 (Vue 3 + Tailwind)**
* 实现全局快捷键唤醒机制，搭建无头 UI 搜索输入框与虚拟列表渲染组件，确保文件名检索响应时间 $\le 20ms$。
* 编写输入法解析器，支持对 `name:`, `content:`, `ext:` 及纯正则语法的实时高亮与错误降级处理。
* 当前进展：已补齐 Vue 3 + Vite + Tailwind + Pinia 前端基座，完成 Raycast 风格搜索台、配置化 `name:` / `content:` / `ext:` / 正则输入解析、错误降级 chip、虚拟列表窗口计算、字段指令过滤、Tauri v2 `search_file_names_command` 检索接入、配置化全局快捷键唤醒窗口动作，以及 10,000 条文件名索引窗口的 $\le 20ms$ 基准测试；`npm test`、`npm run build`、`cargo test` 与 `cargo build` 已通过。


* [x] **Task 3.2：集成 QuickLook 预览与状态流转管理 (Pinia)**
* 开发侧边栏沉浸式预览面板：针对命中内容，自动滚动至匹配段落并提取前后 500 字符进行高亮截断展示。
* 通过 `composables` 封装与 Tauri 层的异步命令交互，处理 `Init`、`Building`、`Watching`、`Suspended` 核心状态机的 UI 反馈。
* 当前进展：已完成 QuickLook 预览 composable，支持从真实文件全文中提取命中位置前后 500 字符、返回局部高亮范围与前后截断标记；已完成 Tauri `read_preview_content_command` 文件内容读取、配置化大小阈值保护、Pinia 预览缓存、选中结果异步预览加载、预览面板自动滚动至高亮段落，以及 `Init`、`Building`、`Watching`、`Suspended` 状态到 UI 反馈模型；`npm test`、`npm run build`、`cargo test` 与 `cargo build` 已通过。



#### Phase 4：测试与联调验证 (Testing & Validation)

* [x] **Task 4.1：执行极端资源约束与边界数据注入测试**
* 模拟低电量与高负载环境，断言系统能否在 1 秒内准确挂起大批量 PDF/Markdown 内容解析任务并抛出正确的状态码。
* 注入不闭合的正则表达式及超过 50MB 的乱码文件，验证系统是否能平滑降级为模糊查询或标记为 `Parse_Error` 而不引起崩溃。
* 当前进展：已补齐资源挂起联动测试，验证 Suspended 状态下内容解析队列在 1 秒内阻断但仍保留文件名更新；已补齐乱码/非 UTF-8 内容解析状态，`Parse_Error` 从 `indexer_policy.json` 配置加载并由预览命令稳定返回；不闭合正则与超 50MB 文件边界测试保持通过。`npm test`、`npm run build`、`cargo test`、`cargo fmt --check` 与 `cargo build` 已通过。
