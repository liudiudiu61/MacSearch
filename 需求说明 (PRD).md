### 0. 💡 PM 的需求洞察与重构（核心亮点）

* **用户原始意图：** 开发一款 macOS 平台上的本地工具，将极速文件名搜索和文件内部文字搜索功能合二为一，填补 Mac 上类似 Everything 和 AnyTXT 的空缺。
* **AI 深度推演与补全：**
* **生态壁垒突围：** macOS 自带的 Spotlight 其实包含了文件名和内容搜索，但用户依然觉得不够用。为什么？因为 Spotlight 响应不够极致、对代码/特殊文档格式解析弱、且缺乏正则和精准路径控制。因此，我们的核心不是“做个搜索”，而是做个“精准且可控的开发者/专业向检索器”。
* **资源调度与降级机制：** Everything 在 Windows 上极快是因为读取了 NTFS 的 USN 日志，而内容检索（AnyTXT）极其消耗 CPU 和 IO。如果把两者融合，必须补全“资源嗅探引擎”（判断 Mac 是否插电、CPU 温度），否则初始建立全盘内容索引时会让 Mac 风扇狂转，直接导致用户卸载。
* **权限沙盒闭环：** macOS 对隐私保护极度严格。必须补全一整套引导用户开启“完全磁盘访问权限 (Full Disk Access)”的交互闭环，否则工具根本扫不到任何文件。

### 1. 🎯 产品全局定义

* **产品定位：** 一款专为 macOS 打造的、毫秒级响应、支持深度内容穿透的本地硬核搜索器。
* **目标用户与核心场景：**
* **研发工程师/研究员/内容创作者** 在 **高压工作、面对海量本地文件和代码库** 的环境下，遇到 **Spotlight 搜不到特殊格式代码片段、无法限制特定目录搜索** 的问题，期望达到 **毫秒级定位目标文件、支持正则匹配并直接预览文件内高亮文本** 的目的。


* **业务目标：** 提供 10 倍于自带 Spotlight 的搜索精度和速度，打造 Mac 生产力工具圈的口碑爆款，为后续（可能的）Pro 版（如 OCR 识图搜索、网盘挂载搜索）建立强大的本地用户基座。

### 2. 🗺️ 业务架构与状态流转

* **业务全景架构：**
* **C端交互层 (UI Layer)：** 极简全局呼出唤醒条 (类似 Raycast/Alfred)、搜索结果高亮列表、QuickLook 沉浸式预览面板。
* **核心检索引擎 (Query Engine)：** 多模态检索引擎（区分文件名快速查询与全文倒排索引查询）、正则表达式解释器、布尔逻辑 (AND/OR/NOT) 处理器。
* **底层索引构建基座 (Indexer Daemon)：**
* **轻量监控器：** 基于 macOS `FSEvents` API 的文件目录实时变动监听。
* **文本解析器：** 多格式文档解包器 (TXT, Markdown, PDF, Office, JSON/XML代码等)。


* **系统控制中心 (Control Center)：** 电源与系统状态嗅探器、隐私权限管理模块、自定义规则引擎。


* **核心主流程 (Happy Path)：**
全局快捷键唤醒界面 -> 混合输入搜索词 (如 `ext:pdf 架构设计`) -> 引擎并发请求两套索引 -> UI 毫秒级展示文件名匹配结果，并异步追加内容匹配结果 -> 上下键切换 -> 右侧实时预览命中内容并高亮 -> 按 Enter 打开或按 Cmd+C 复制内容。
* **关键状态机（本地索引构建状态）：**
* `Init (待初始化)`：首次安装，提示获取“完全磁盘访问权限”。
* `Building (全量建库)`：插电状态下，遍历设定的目录进行分词和倒排索引构建。
* `Watching (实时监听)`：基于 FSEvents，静默监听文件增删改，秒级更新索引。
* `Suspended (挂起)`：检测到电池供电 < 20% 或 CPU 负载 > 80% 时，主动暂停内容提取，仅保留文件名更新。

### 4. 🛡️ 非功能性与扩展性需求

* **性能与并发：**
* **文件名检索响应时间** 要求 $\le 20ms$（达到肌肉记忆般的即时感）。
* **内容检索首屏结果响应** 要求 $\le 200ms$。
* **内存控制 (RAM Constraint)：** 倒排索引库可能非常大，绝对不能全量读入内存。必须采用内存映射文件 (mmap) 技术或底层优化的 SQLite 查询，将常驻内存控制在 $150MB$ 以内。


* **系统扩展性设计（Plugin Architecture）：**
* 内容解析层需要做接口化设计 (Parser Interface)。第一版自己写 PDF/Word 解析，后续可开放标准输入输出协议，让社区写 Python/Go 脚本来解析特殊的专业格式（如 CAD 文件图层名、设计软件画板名）。


* **合规与安全：**
* **数据绝对本地化：** 承诺 100% 离线运行。绝不上传任何索引数据到云端，所有跨设备同步（若未来开发）均需基于端到端加密 (E2EE)。
