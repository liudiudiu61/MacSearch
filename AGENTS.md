# 核心技术栈
- 前端: Tauri v2 + Vue 3 (Setup/TS) + Tailwind + Pinia
- 后端: FastAPI + Pydantic v2 + SQLModel + PostgreSQL (云) + SQLite (端)

# 绝对红线 (Zero Hardcoding)
1. 严禁在前后端写死任何业务规则、黑名单路径、正则或状态码，必须从配置字典或 DB 加载。
2. 数据模型必须保持物理标识与专业描述的强制融合，父子级关联必须通过外键/级联完整定义。
3. Vue 3 必须使用 `<script setup>` 和 OpenAPI 自动生成的 TS 类型，禁止手动拼写 API 接口。复杂逻辑一律抽离至 `composables`。

# 核心编程行为约束
- **谋定而后动**: 编写代码前，必须先输出简短的假设和逻辑选项，严禁默默猜测模糊需求。
- **极简与精准**: 仅编写满足当前需求的最少代码，禁止过度设计。
- **目标驱动执行**: 必须按照 "设计验证标准 -> 实施编码 -> 确认通过" 的闭环执行。多步任务必须先列出 Checklist。
- **SSE/异步优先**: 所有耗时操作(如全文解析、AI调用)必须采用异步处理或 SSE 流式响应，禁止阻塞主线程。

# 动态状态流转规则（State Tracking Rules）
在对话中，当用户回复"完成 Task X"或"执行 Task X"时，需要：​
- **分析该 Task 完成后的系统状态。**
- **重新打印上述《结构化开发计划》，将对应的状态标记为 [x]（已完成）或 [-]（部分完成/受阻）。**
- **针对下一个即将执行的 Task，输出具体的代码实现建议或提供风险警示。**
